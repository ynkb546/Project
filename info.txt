1. Цель разработки приложения
Приложение создано для автоматизации процесса управления товарными запасами на основе исторических данных о продажах.

Главная цель — найти баланс между объемом закупок и спросом, чтобы максимизировать прибыль.

Приложение помогает избежать потери средств в излишних запаса,убытках от истечения срока годности
 и одновременно не допустить упущенной выгоды из-за дефицита.


 2. Решенные задачи
Автоматическое разделение товаров на штучные и весовые для корректной аналитики.

Группирование исторических данных по неделям.

Моделирование продаж по нескольким сценариям, чтобы выявить наиболее прибыльную стратегию поставок.

Реализована логика списания партий товара: сначала продаются самые старые партии,
а если срок годности истекает — товар списывается в убыток.

Расчет стоимости хранения, выручки от продаж, убытков от списаний и упущенной выгоды.

Построение наглядных диаграмм.


3. Используемые средства
Проект написан на языке Python (3.13.12).
Библиотеки:

pandas (3.0.3): Выбран для загрузки и обработки табличных данных.

numpy (2.4.4): Использован для математических вычислений

matplotlib (3.10.9) и seaborn (0.13.2): Выбраны для построения статичных аналитических графиков

plotly (6.7.0): Использован для интерактивных графиков.

tkinter Выбран для создания графического интерфейса пользователя

4. Что было добавлено с предзащиты

графический интерфейс : Теперь это полноценная программа с навигационным меню и всплывающими диалоговыми окнами.

прогнозирование: Добавлен совершенно новый функционал — теперь приложение может принимать новые данные от пользователя и прогнозировать поставку на следующую неделю.

Таблица сравнения стратегий: Добавлена возможность сравнивать различные квантильные квантили.


5. Структура:

state
Отвечает за хранение текущих настроек и результатов лучшей стратегии.

df: DataFrame, в котором хранится итоговая таблица с прогнозом (история симуляции).

batches: Список словарей.Каждый словарь внутри — партия товара с её остатком и сроком годности.

last_date: Дата последней смоделированной недели

q_value: Рассчитанное значение запаса для лучшего квантиля.

shelf_life, storage_cost, price, product_name: Сохраненные настройки, которые ввел пользователь.

metrics
Хранит заранее рассчитанные данные:

total_packs, total_weight: Общее количество проданных пачек и развесного товара .

top_packs, top_weight: Готовые датафреймы с Топ-5 товаров .

plotly_weekly: Данные, сгруппированные по неделям, для отрисовки в интерактивном графике.

table_name: Имя загруженного файла.


center_window(window, width, height)
Функция нужна для того, чтобы окна появлялись я опеделенном месте на экране.

show_treeview_window(root, df, title)
Отвечает за отрисовку таблиц. Используется для вывода отчетов по неделям, матрицы стратегий и финального прогноза.

run_quantile_simulation(prod_data, target_q, shelf_life, storage_cost, unit_price)
Функция прогоняет данные продаж через выбранную квантильную стратегию.

add_interactive_week(root)
Позволяет программе симмулировать недели продаж и поставок.

main()
Функция запускающая всю программу.
